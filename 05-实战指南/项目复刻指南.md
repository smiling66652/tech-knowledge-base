## 项目1：Docker镜像转存服务

### 基本信息
- **来源UP主**：技术爬爬虾
- **难度**：⭐⭐
- **时间**：30分钟
- **Stars**：15.9k

### 复刻前准备
- GitHub账号
- 阿里云账号（开通容器镜像服务）
- 基本的Git操作能力

### 复刻步骤

```
Step 1: Fork仓库
  → https://github.com/tech-shrimp/docker_image_pusher
  → 点击Fork按钮

Step 2: 配置阿里云
  → 登录阿里云 → 容器镜像服务
  → 创建命名空间（如：my-mirrors）
  → 设置仓库固定密码

Step 3: 配置GitHub Secrets
  → 进入Fork的仓库 → Settings → Secrets and variables → Actions
  → 添加以下Secrets:
    - ALIYUN_REGISTRY_USER: 你的阿里云用户名
    - ALIYUN_REGISTRY_PASSWORD: 你的阿里云密码
    - ALIYUN_NAMESPACE: 你的命名空间

Step 4: 修改镜像列表
  → 编辑仓库中的配置文件
  → 添加你想转存的Docker镜像

Step 5: 运行
  → Actions → 选择工作流 → Run workflow
  → 等待执行完成

Step 6: 验证
  → 在阿里云控制台查看转存的镜像
  → 在本地拉取测试：docker pull registry.cn-xxx.aliyuncs.com/my-mirrors/nginx:latest
```

### 进阶改造
- 添加自动定时转存（每月更新镜像）
- 添加镜像版本监控（有新版本自动通知）
- 添加多区域同步

- **时间**：2-3小时

- Docker已安装
- 基本的n8n操作了解

### 复刻步骤

```
Step 1: 部署n8n
  → 创建docker-compose.yml
  → docker-compose up -d
  → 访问 http://localhost:5678

Step 2: 创建第一个工作流
  → 添加"Manual Trigger"触发器
  → 添加"HTTP Request"节点（获取数据）
  → 添加"AI Agent"节点（AI处理）
  → 添加"Webhook"节点（发送结果）
  → 连接所有节点

Step 3: 测试运行
  → 点击"Execute Workflow"
  → 检查每个节点的输出

Step 4: 设置定时触发
  → 将触发器改为"Cron"节点
  → 设置执行时间（如每天8:00）
```

## 常见问题汇总

### Q: Fork的项目和原项目有什么区别？
A: Fork是在你自己的GitHub账号下创建一个副本，你可以自由修改，不影响原项目。

### Q: 如何获取阿里云/GCP的API Key？
A: 各平台官网注册账号后，在控制台的API管理页面创建。

### Q: Docker部署失败怎么办？
A: 检查Docker是否正常运行，端口是否被占用，网络是否通畅。

### Q: 遇到不理解的代码怎么办？
A: 1. 用Claude/ChatGPT解释代码 2. 查看项目文档 3. 搜索CSDN/GitHub Issues

### Q: 如何与原UP主互动？
A: 在B站评论区留言，在GitHub提Issue，在CSDN文章下评论。
