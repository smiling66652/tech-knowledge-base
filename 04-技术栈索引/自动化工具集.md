## 一、n8n 工作流自动化

### 核心信息

| 维度 | 说明 |
|------|------|
| **官网** | https://n8n.io |
| **开源** | 是（可自托管） |
| **费用** | 免费（自托管）/ $20/月起（云服务） |
| **UP主关联** | 小in分享 |

### 核心功能

```
触发器（Trigger）
├── Webhook     → HTTP请求触发
├── Cron        → 定时触发
├── Manual      → 手动触发
├── Email       → 邮件触发
└── Database    → 数据库变更触发

操作节点（Node）
├── AI Agent    → AI处理（内置）
├── HTTP        → API调用
├── Database    → 数据库操作
├── File        → 文件读写
├── Condition   → 条件判断
├── Loop        → 循环处理
└── 400+ 内置集成
```

### 适用场景

| 场景 | 说明 |
|------|------|
| 数据采集+推送 | 定时采集网页数据，AI分析，推送到微信/邮件 |
| AI工作流 | 链接多个AI服务，构建智能处理管道 |
| 内容发布 | 自动生成内容并发布到多平台 |
| 监控告警 | 监控网站/服务状态，异常时推送通知 |

## 二、GitHub Actions CI/CD

### 核心信息

| 维度 | 说明 |
|------|------|
| **官网** | https://github.com/features/actions |
| **费用** | 公开仓库免费，私有仓库2000分钟/月 |
| **UP主关联** | 技术爬爬虾 |

### 核心概念

```
Workflow（工作流）
├── Trigger（触发器）
│   ├── push / pull_request
│   ├── schedule（cron定时）
│   └── workflow_dispatch（手动）
├── Job（任务）
│   ├── runs-on（运行环境）
│   └── steps（步骤列表）
└── Step（步骤）
    ├── uses（使用Action）
    └── run（执行命令）
```

### 实用配置

#### 定时运行

```yaml
on:
  schedule:
    - cron: '0 0 * * *'  # 每天UTC 00:00
```

#### Python项目

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt
      - run: python main.py
```
